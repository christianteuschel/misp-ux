# MISP User Stories

Source: https://www.circl.lu/doc/misp/user-stories/ (last modified: 2024-10-02)

| User Story | Example Workflows |
| --- | --- |
| As a lead threat intelligence analyst, I want to lead a team focused on hunting down threats so that I can prevent attacks against ICT infrastructures and organizations | - Monitor what teams are up to in real-time using the Live Dashboard |
| As a threat analyst, I want to research, analyze and reverse engineer malware so that I can know how to counter it | - Attach and download files and malware samples from events<br>- Search for hashes/IPs/domains/URLs from malware events, or add malware sample hashes to an event<br>- Analyse observables and malware collected during an incident (e.g. domain name, IP addresses) by checking whether observables are IoCs or false positives using the correlation graph and expansion modules<br>- Enrich malware events by querying data sources external to MISP using modules<br>- Perform dynamic malware analysis correlations<br>- Submit events with malware samples to analysis tools (e.g. VirusTotal, VMRay) for further analysis, then extend MISP with the results |
| As a lead threat intelligence analyst, I want to convert threat data into actionable threat intelligence so that I can improve security posture | - Import data from external sources<br>- Add feeds<br>- Contextualise events and attributes using tags, taxonomies and galaxies |
| As a threat analyst, I want to exchange threat information with third parties so that we can gain shared situational awareness | - Set up different models of distribution on a MISP instance<br>- Sync events and attributes between instances<br>- Use filtering functionalities to meet an organisation's sharing policy<br>- Share information, pentest information, malware samples, and vulnerabilities internally and externally<br>- Use the achievements widget to add gamification to information sharing |
| As a threat analyst, I want to monitor threats and access live data so that I can manage threats before they cause major damage | - Import lists of indicators and check if the IOCs are present in feeds<br>- Monitor statistics and sightings using widgets<br>- Show live data and stats from one or more MISP instances via the Dashboard<br>- Process information in real-time when it's updated, created, or published by instances by integrating with ZMQ<br>- Use sightings to notify an instance about activities related to an indicator |
| As a threat analyst, I want to aggregate and compare indicators from various sources so that I can connect the dots between various threats | - Join communities and subscribe to feeds<br>- Add events and assign events to specific feeds<br>- Correlate indicators using MISP's automated correlation engine<br>- Use the overlap feed analysis available in MISP<br>- Link events and attributes using the correlation graph<br>- Analyse and gain more information on attributes using modules<br>- Link events with malware, threat actors etc. using galaxies (e.g. ATT&CK) |
| As a threat analyst, I want to have a structured database of threat data that I can use to perform lookups/queries when investigating new threats | - Store information in a structured format using STIX<br>- Import unstructured reports using the free-text import tool<br>- Use MISP as a centralised hub for security and fraud threat intel by aggregating indicators from OSINT and commercial feeds<br>- Remove false positives and duplicates<br>- Score indicators based on sightings and other metrics<br>- Import/integrate feeds or threat intelligence from third parties<br>- Select and import events<br>- Look for correlations between events using the correlation graph<br>- Build filtered subsets of the data repository for feed creation<br>- Preview and correlate feed data directly for evaluation |
| As a threat analyst, I want to contextualise and enrich raw threat data so that I can produce actionable intelligence | - Understand attacker TTPs by using taxonomies to link events<br>- Categorise risks and incidents using galaxies and taxonomies<br>- Quickly classify information using tag collections<br>- Contextualise sightings with information on the source<br>- Enrich IDS exports with tags to fit your NIDS deployment<br>- Decay attributes and score indicators using sightings (reported by IDSes)<br>- Describe and visualise complex scenarios using MISP's richer data structure<br>- Allow advanced combination of attributes using MISP objects |
| As a threat analyst, I want to investigate threats so that I can protect computer systems from attacks | - Find relevant data for investigations from MISP communities; preview new events and alerts from multiple sources such as email reports, CTI providers, and SIEMs<br>- Query a MISP instance for events that include a given IOC; browse events, attributes, objects, tags, and galaxies<br>- Create events, add IoCs (attributes), and contextualise using tags<br>- Pivot an event into its attributes, objects, tags, galaxies, and/or related events<br>- Explore further details from galaxies and related events<br>- Categorise available related information within the ATT&CK framework<br>- Query tools to check if certain MISP indicators have been observed, then import sighting details back to MISP events<br>- Prioritise threats using sightings collected from users, scripts and IDSes<br>- Decay/expire indicators using sightings reported by users, scripts and IDSes<br>- Launch lookups from MISP against SIEMs as part of an investigation<br>- Correlate network forensic flows from several tools |
| As a SOC team, we want to ingest, analyse, store and make connections between threat data so as to discover potential threats | - See connections between events using the correlation graph<br>- Import CVEs and vulnerabilities (e.g. from MetaSploit) and contextualise them<br>- Contextualise CVEs using events from articles/reports<br>- Convert CVE information into a feed<br>- Pull shared CVE feeds<br>- Combine collected data with your MISP data set for correlation<br>- Share correlated info to the team using the export function or API search<br>- View current threats and activity, historical and geolocalised information using the MISP Dashboard |
| As a junior SOC analyst, I want to enrich alerts so that I can "punch above my weight" and make connections that would have otherwise required more experience | - Create events, add/import observables<br>- Use Cortex and its analysers to gain insight<br>- Leverage tags, sightings, and previously-seen observables to feed your threat intelligence<br>- Export IOCs to MISP instances after investigations are complete<br>- Integrate MISP with Maltego to generate visualisations of data<br>- Integrate MISP with Elastic to access threat data without the complexities of the MISP interface<br>- Push attributes from MISP to Elastic and have a representation with graphs, as an alternative to the MISP Dashboard<br>- Create taxonomies using the taxonomy editor<br>- Contextualise data using taxonomies, clusters and galaxies |
| As a SOC analyst, I want to customise risk feeds to ignore or downgrade alerts that do not match organisation/industry-specific criteria, so that I can focus on relevant alerts | - Filter incidents based on taxonomies (e.g. the VERIS country taxonomy to indicate countries affected by an incident)<br>- Normalise external input and feeds in MISP (e.g. feed importer)<br>- Compare feeds before import to find similarities and false positives<br>- Evaluate the quality of information before importing it (warning-list lookups at feed evaluation) |
| As a SOC analyst, I want to share real-time information pertaining to new or existing cases/observables to team members so that we can collaborate on investigations simultaneously | - Control threat sharing using distribution settings: sharing group, community-only, connected communities, all communities<br>- Share sensitive and confidential events using the sharing group functionality<br>- Measure the impact of an incident using taxonomies based on NIS/OES impact criteria<br>- Export and share sightings in ATT&CK sightings format to give insights on TTPs and frequency of usage |
| As a SOC analyst, I want to rule out false positives so that I can focus on significant threats | - Weed out false positives using warning lists<br>- Crowd-source data validation from the community<br>- Filter indicators based on specific criteria<br>- Receive information on false positives using collaborative tools (proposals, sightings) |
| As a threat analyst, I want to remove false positives, filter and prioritise alerts so that I can focus on what really matters to my organisation | - Evaluate the quality and freshness of indicators using decaying models<br>- Enforce warning lists to exclude events with certain attributes<br>- Enable warning lists to alert for certain issues<br>- Classify information (add/remove tags) based on their score or visibility via sightings<br>- Use tags to set events or attributes for further processing by external tools (e.g. VirusTotal auto-expansion using Viper)<br>- Notify an instance about activities related to an indicator via sighting<br>- Limit NIDS exports and improve rules using sightings<br>- Filter indicators based on specific criteria<br>- Filter out relevant data when feeding protective tools |
| As a security analyst, I want to unravel the inner workings of a malicious file, phishing email or domain so that I can prevent attacks | - Integrate MISP with a Security Incident Response Platform (e.g. TheHive)<br>- Import indicators from MISP into the SIRP for further analysis |
| As a security analyst, I want to create blacklists/whitelists (e.g. of domains) so that I can protect customers from malicious activity | - Import threat data into MISP from synced servers and label using taxonomies<br>- Enable warning lists, and exclude attributes that exist on the warning lists<br>- Create lists with preferred attributes and export in an easily accessible format such as CSV |
| As a security analyst, I need a real-time overview of threat information so that I can quickly glance at important metrics | - Integrate ZMQ to access a dashboard showing live data and stats<br>- Monitor ongoing trends based on interests using the EventStream widget<br>- Monitor activity in real-time on the MISP dashboard by subscribing to ZMQ feeds<br>- View immediate contributions made by organisations from MISP's live dashboard<br>- Find threats within your constituency using the MISP Geolocalisation Dashboard<br>- Get geospatial threat information from specific regions using the Geolocalisation Dashboard |
| As a security analyst, I want to automate repetitive tasks related to data normalisation, importation, aggregation and enrichment so that I can have more time for threat analysis | - Automate tasks using PyMISP<br>- Use PyMISP for scripted processing of events and attributes |
| As a security analyst, I want to collaborate with other analysts within and outside my organisation's sector so that we can support one another | - Build or join communities to exchange specific data structures<br>- Share real-time analysis of an incident<br>- Propose modifications to someone else's analysis using proposals |
| As a security analyst, I want to triage and prioritise alerts so as to avoid alert fatigue | - Evaluate the quality and freshness of indicators using decaying models<br>- Weed out false positives using warning lists<br>- Enable warning lists to alert for critical issues<br>- Filter indicators based on specific criteria<br>- Score indicators based on user sightings, including negative sightings and expiration sightings<br>- Classify information (add/remove tags) based on their score or visibility via sightings |
| As an incident responder, I want to get an up-to-date picture of the threat landscape so that I can prepare for threats in advance | - Describe the impact of a threat using taxonomies (e.g. using the VERIS timeline taxonomy to indicate the duration of an incident)<br>- Classify data to gain insight into the threat landscape<br>- Classify data so IDSes can alert on a rule<br>- Integrate ZMQ to have a dashboard showing live data and statistics<br>- Integrate ZMQ to process information in real-time when it's updated, created, or gathered in MISP |
| As an incident responder, I want to identify and respond to incidents so that I can reduce the impact and severity of an attack | - Report false or true positives using the sighting mechanism, based on an incident investigation<br>- Decay indicators to guarantee their quality |
| As an incident responder, I want to receive early warnings and alerts about threats/incidents so that I can retaliate before they cause any harm | - Receive correlated threat intel from sharing groups and communities<br>- Monitor MISP feeds for alerts<br>- Preview new events and alerts from multiple sources<br>- Automate import/export of IoCs to/from protective or detection tools like IDSes and IPSes<br>- Dispatch notifications when certain events are created or modified using the alert feature<br>- Create filter rules based on personalised uses; restrict alert messages by tags, publishing organisation or other metrics |
| As an incident responder, I want to store information identified during an incident investigation so that I can perform lookups/queries against the historical database during future incidents | - Use a MISP instance as a database of events representing incidents; store incident response data in a structured manner<br>- Represent indicators using attributes (e.g. network indicators such as IP addresses, system indicators such as strings in memory)<br>- Combine OSINT and your own intelligence<br>- Create events made up of indicators (attributes) and leverage these as a threat data feed<br>- Modify events representing incidents to enable monitoring over time<br>- Add object types to describe incidents<br>- Monitor indicators for relevancy using sightings<br>- Expire indicators depending on personalised objectives<br>- Pull events from indicator lists to perform lookups against SIEMs<br>- Use indicators to check logs and verify if you're affected by a threat<br>- Correlate indicators with actual incidents to get more information<br>- Integrate MISP with IR tools (e.g. TheHive) to analyse observables and import/export events<br>- Perform large-scale bulk data/traffic analysis and correlation using SightingsDB |
| As an incident responder, I want to export and feed data between security tools so that I can enhance their functionalities | - Export data from MISP to feed protective/detective tools and early warning systems; supported formats include IDSes/IPSes (Suricata, Bro, Snort), SIEMs (CEF), host scanners (OpenIOC, STIX, CSV, YARA), analysis tools (Maltego), and DNS policies (RPZ)<br>- Feed MISP using automatic tools (e.g. sandbox analysis, analyst workbench)<br>- Pull events from feeds or indicator lists to perform lookups against SIEMs<br>- Subscribe to ZMQ pub-sub to get published events for use in lookup processes<br>- Match attributes against SIEMs using the lookup expansion module<br>- Import activities from a SIEM (e.g. Splunk lookup validation or false-positive feedback), NIDS or honeypot devices<br>- Post sightings from IDSes, IPSes, SIEMs back to MISP<br>- Use sightings to improve NIDS rule-sets<br>- Generate IDS and NIDS rules automatically or manually using IoCs<br>- Feed data to honeypots to generate blocklists and DNS RPZ zones<br>- Consume correlated results in SIEMs using the API<br>- Search indicators in real-time into a SIEM using MISP ZMQ<br>- Submit large sets of IoCs from MISP into SIEMs using PyMISP<br>- Import indicators into MISP from other tools (SIEMs, IDSes) and be notified when those indicators appear again |
| As a CSIRT, we want to exchange and discuss information related to incidents and associated risks so that we can collaboratively respond to incidents | - Build communities to exchange specific data structures<br>- Discuss non-event-related topics in forums<br>- Add comments to events (which may represent an incident)<br>- Contact a reporter (e.g. another CSIRT) via email (encrypted, anonymously or not) to discuss commercially sensitive information related to an incident |
| As a CSIRT, we want to interact with threat data in various ways during the threat investigation and incident response process | - View events, indicators and feeds<br>- Search and filter the data set<br>- Classify, contextualise and correlate data<br>- Download the viewed data in various formats<br>- Interact with MISP data using other tools in the MISP ecosystem (e.g. MISP Workbench, Viper, MISPego) |
| As a CSIRT, we want to coordinate with team members and other organisations so that we can avoid duplication of work | - Create and manage sharing groups between sectors<br>- Join existing communities or sharing groups<br>- Create and exchange events and indicators<br>- Propose changes to existing analysis or reports<br>- Enhance an analysis with additional information using Extended Events<br>- Report sightings as false-positive or true-positive<br>- Contribute to threat intel feeds and analyse overlapping data |
| As a CSIRT, we want to share incident information and discuss risks with other team members so that we can collaboratively perform incident analysis | - Create, modify, delete and exchange events and indicators<br>- Modify distribution settings to exchange individual incidents and ensure confidentiality<br>- Use taxonomies and galaxies to classify data before exchange (e.g. NATO classification for confidentiality, threat-level taxonomy for risk)<br>- Edit, visualise, and share reports using Event Report<br>- Incorporate reports from information sources using the Event Report module<br>- Share indicators derived during incident response<br>- Correlate and enrich data derived during incidents<br>- Coordinate with affected parties using MISP's collaborative tools (proposals, sightings, emails) |
| As a fraud analyst, I want to investigate financial threats so that I can help financial institutions and consumers prevent financial fraud | - Join communities and receive shared IOCs<br>- Subscribe to feeds and get IOCs in an easily accessible format<br>- Access lists and public feeds of malicious domains (e.g. phishing sites) and threats<br>- Use indicators to check logs and verify if you're affected by a threat<br>- Gather information related to a phishing site and create events<br>- Integrate MISP with Maltego to visualise the full ATT&CK framework |
| As a fraud analyst, I want to blend updated threat intel with anti-fraud tools so that I can prevent fraud in real-time | - Feed data from MISP to fraud prevention tools<br>- Report sightings to MISP from fraud prevention tools |
| As a fraud analyst, I want to collaborate with analysts from other institutions so that we can gain shared situational awareness | - Implement a MISP instance and join relevant communities<br>- Publish fraud perpetrators for others to see<br>- Exchange events containing fraud information (e.g. a bank account number)<br>- Use shared fraud data to feed firewalls and blocklists<br>- Warn of false positives by alerting for invalid financial indicators<br>- Give more credibility to indicators by reacting to event attributes (sightings)<br>- Get feedback from the community on the quality of indicators (sightings) |
| As a customs and border control agent, I want to facilitate the flow of legal immigration and goods while preventing the illegal trafficking of people and contraband so that I can ensure homeland security | - Create or join sharing groups and communities<br>- Share information (e.g. travel documents, biometric information) between border control agencies using MISP<br>- Categorise data using predefined types such as PNR (passenger name records)<br>- Share information and involve experts for the identification of smuggled goods<br>- Perform anonymised lookups against exported data sets (e.g. offline border control checks) |
| As a law enforcement officer, I want to investigate digital crimes and threats so that I can apprehend criminals | - Access information sharing communities<br>- Get indicators and actionable information from CSIRT/CERT networks or researchers<br>- Exchange information with other officers via sharing communities<br>- Exchange and store incident information on MISP, enabling the system to act as a forensic tool over time |
| As a law enforcement officer, I want to collect and verify evidence of digital crimes so that I can bootstrap my DFIR cases | - Collect indicators from shared events<br>- Propose changes to existing analysis or reports<br>- Enhance existing events with additional pieces of evidence using Extended Events<br>- Exchange analysis and reports of digital forensic evidence<br>- Correlate indicators corresponding to forensic pieces of evidence<br>- Import Mactime timelines to describe forensic activities on an analysed file system<br>- Describe forensic analysis cases using object templates<br>- Create, modify and visualise the timeline of events<br>- Share analysis and reports of digital forensic evidence<br>- Report sightings as false-positive or true-positive |
| As a cybersecurity consultant, I want to provide structured threat intelligence to cross-sector partners with diverse requirements so that I can secure their infrastructure | - Implement an instance and join relevant communities<br>- Integrate MISP with an organisation's existing solutions using the API<br>- Exchange events containing indicators<br>- Set up distribution levels to ensure confidentiality during threat sharing<br>- Sync between untrusted and trusted networks using feed support<br>- Notify the community about activities related to an indicator using sightings<br>- Score indicators based on user sightings, including negative sightings and expiration sightings<br>- Propose updates to an event owner or indicate a sighting<br>- Share attacker techniques via integration with ATT&CK<br>- Set an attribute for detection tools using the IDS flag |
| As a cybersecurity specialist, I want to anonymously publish threat intel so that I can protect the identity of people who don't want to be associated with the information | - Pseudo-anonymously publish data using Event Delegation |
| As a cybersecurity specialist, I want to investigate threats so that I can remediate and prevent cyber attacks | - Query an instance for events that include a given IOC<br>- Explore more details from galaxies and related events<br>- Categorise related information within the MITRE ATT&CK framework |
| As a security analyst, I want to access threat data so that I can use it to support my research | - Contextualise indicators using categories, taxonomies and galaxies<br>- Reinforce an analysis using correlation features (e.g. do other analysts have the same hypothesis?)<br>- Confirm a specific aspect using correlation features (e.g. are the sinkhole IP addresses used for one campaign?)<br>- Verify if a threat is new or unknown in your community using correlation features |
| As a security analyst, I want to access updated threat data so that I can build protection in real time | - Monitor feeds for recent indicators<br>- Monitor activity in real-time on the MISP dashboard by subscribing to ZMQ feeds<br>- Process information in real-time when it's updated, created or gathered using ZMQ |
| As a risk analyst, I want to identify and predict risks to my organisation so that I can improve the organisation's security posture and situational awareness | - Use a MISP instance as a database of events representing threats<br>- Classify risks using taxonomies and galaxies<br>- Generate statistics from your MISP instance to deduce the current operational status, risk posture, and threats to the cyber environment<br>- Monitor trends and adversary TTPs using MISP Dashboard and built-in statistics |
| As a risk analyst, I want to present risk data to stakeholders in various formats (depending on their technical ability), so that I can justify the need for risk-mitigating strategies | - Show trends within the sector/geographical region using MISP Dashboard and built-in statistics<br>- Turn MISP data into explorable graphs or timelines representing activity or events<br>- Export data from MISP in various formats<br>- Share reports along with actionable data using Event Reports |
| As a disinformation researcher, I want to identify indicators associated with a specific operation or campaign so that I can help track and mitigate threats | - Monitor MISP feeds for indicators<br>- Find relationships between indicators using correlation |
| As a disinformation researcher and journalist, I want to investigate information campaigns so that I can report whether there is or isn't disinformation or misinformation | - Compare external feed information with already-available information<br>- Analyse the connections between incident objects<br>- Map data with AMITT (embedded in MISP) to understand threat actor capabilities<br>- Generate events that can be shared directly, via email or MISP<br>- Add object types (e.g. for common social media platforms), relationship types, and taxonomies (e.g. DFRLab's Dichotomies of Disinformation) to describe indicators and events<br>- Generate and share information operations data in MISP JSON or STIX format<br>- Classify events with AM!TT techniques using the inline AM!TT Navigator<br>- Track disinformation techniques using the AMITT galaxy<br>- Integrate MISP with TheHive for case tracking<br>- Describe additional disinformation cases using object templates |
| As a disinformation researcher, I want to connect with other researchers and responders so that we can collaboratively verify disinformation in articles, videos, images and sources | - Join a disinformation community<br>- Notify the community about activities related to an indicator<br>- Score indicators based on user sightings<br>- Corroborate a finding using correlation features (e.g. is this the same campaign?) |
| As a disinformation researcher, I want to collaborate with other researchers and responders so that we can collectively stop disinformation campaigns | - Browse and join disinformation communities (e.g. CogSec Collab MISP)<br>- Contextualise data using tags, taxonomies and galaxies<br>- Describe information campaign indicators and events using taxonomies (e.g. DFRLab Dichotomies of Disinformation)<br>- Find relationships between indicators using correlation<br>- Describe misinformation tactics/techniques using the AM!TT framework (galaxy)<br>- Include relevant techniques found in a report or sighting in misinformation event data using AM!TT Navigator |
| As a data scientist, I want to automate tasks related to data collection, curation, analysis, and visualisation so that I can reduce security analysts' workloads | - Collect, add, update, and search events/attributes/tags using PyMISP<br>- Study malware samples using PyMISP<br>- Write scripts to import additional attributes or IOC data (e.g. hashes from VirusTotal) to build up knowledge on an event<br>- Automatically handle indicators in third-party tools using PyMISP<br>- Integrate MISP with existing infrastructure using PyMISP<br>- Automate the dissemination of threat intelligence and threat data using the API<br>- Generate exports to be ingested into other platforms<br>- Create a range of filtered subsets of the dataset for various protective measures<br>- Write scripts to disable the IDS flag based on the number of false-positive sightings reported<br>- Generate data statistics and send reports via email as CSV files using the API<br>- Feed processed data into IDSes and third-party visualisation tools using PyMISP<br>- Build custom widgets to visualise/track data via the Dashboard<br>- Extend MISP with Python scripts using MISP modules<br>- Auto-discover new modules with their features using the API |
| As a data scientist, I want to collect and analyse data from various sources so that I can prioritise and predict risk | - Aggregate indicators and sightings of all attributes/objects, useful for detecting particular security events or threats<br>- Use PyMISP for scripted processing of events and attributes<br>- Collect data from open data portals using the API<br>- Publish open data and create data sets<br>- Investigate file hashes, malicious website URLs, IP addresses and domain names using shared indicators<br>- Aggregate data sets for security research and threat analysis<br>- Analyse and select threat feeds for incorporation into other tools to hunt known indicators<br>- Indicate if an attribute should be used for detection or correlation actions using the IDS flag<br>- Download data in various formats for ingestion in other tools and for training ML models |
